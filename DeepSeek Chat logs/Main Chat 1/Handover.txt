4. Data Preprocessing Decisions

Step 1 - Handle Absence NA:
Replace NA with 'None' for absence columns

Step 2 - Ordinal Encoding:
Applied to 14 ordinal features with consistent mapping (Ex=5, Gd=4, TA=3, Fa=2, Po=1, None=0)
Special mappings for BsmtExposure (Gd=4, Av=3, Mn=2, No=1, None=0)
Special mappings for BsmtFinType1/2 (GLQ=6, ALQ=5, BLQ=4, Rec=3, LwQ=2, Unf=1, None=0)
Special mapping for Fence (GdPrv=4, MnPrv=3, GdWo=2, MnWw=1, None=0)

Step 3 - MSSubClass Conversion:
Convert from int to categorical (string)

Step 4 - Imputation (Simple):
Numerical features: median imputation (robust to outliers)
Categorical features: most frequent imputation (mode)
Note: Simple imputation may distort distributions. Advanced methods could be explored later.

Step 5 - Scaling:
Method: RobustScaler (less sensitive to outliers than StandardScaler)
Applied to all numerical features after encoding

5. Feature Engineering - Completed

Area Features:
TotalSF = GrLivArea + TotalBsmtSF (combined living space)
TotalBath = FullBath + 0.5*HalfBath + BsmtFullBath + 0.5*BsmtHalfBath (total bathrooms weighted)
TotalPorchSF = sum of all porch types (outdoor living space)

Age Features:
HouseAge = 2010 - YearBuilt (age at sale)
RemodAge = 2010 - YearRemodAdd (years since remodel)
IsRemodeled = (YearBuilt != YearRemodAdd) (binary flag)
IsOnOrBefore1950Remodel = (YearRemodAdd <= 1950) (captures pre-1950 remodel quirk)

Remodel Era Categories:
OnOrBefore_1950: YearRemodAdd <= 1950
Original_No_Remodel: YearRemodAdd == YearBuilt
Remodel_1951_1999: 1950 < YearRemodAdd < 2000
Remodel_2000_Plus: YearRemodAdd >= 2000

Economic Period Features:
IsFinancialCrisis = (YrSold between 2008 and 2009)
IsPostBubble = (YrSold >= 2007)
YearMonth = combined string (dropped later, components extracted)

Binary Presence Flags:
HasBsmt = TotalBsmtSF > 0
HasGarage = GarageArea > 0
HasPool = PoolArea > 0
HasFireplace = Fireplaces > 0

Quality and Ratio Features:
TotalQual = sum of ExterQual, BsmtQual, HeatingQC, KitchenQual, GarageQual
QualPerAge = TotalQual / (HouseAge + 1)
AreaPerRoom = GrLivArea / (TotRmsAbvGrd + 1)

Lot Features:
IsIrregularLot = LotShape in [IR1, IR2, IR3]
IsSevereSlope = LandSlope == Sev

Features NOT Engineered (Decision Log):
MoSold_sin and MoSold_cos removed (seasonality not significant)
Quarter, IsPeakSeason, IsHolidayMonth removed (no evidence of seasonal effects)
YearDecade removed (year fixed effects handled by Year column)

6. Categorical Feature Handling

Categorization by Cardinality:
Low cardinality (2-4 unique): IsOnOrBefore1950Remodel, HasBsmt, IsRemodeled, etc. - One-hot applied
Medium cardinality (5-10 unique): MSSubClass, RemodelEra, Month - One-hot applied
High cardinality (greater than 10 unique): Neighborhood (25), Exterior1st (15), Year (6) - Target encoding deferred to modeling
Special: YearMonth - extracted components, dropped

One-hot encoding applied to all low and medium cardinality categoricals using pd.get_dummies(drop_first=True)

Alignment between train and test:
Add missing columns to test with value 0
Remove extra columns from test

8. Known Issues and Limitations

Simple median/mode imputation (medium severity) - could try KNN imputation, regression imputation, or MICE
Outliers not capped (low severity) - apply Winsorization or capping at 99th percentile
High-cardinality features like Neighborhood (medium severity) - implement target encoding
MSSubClass grouping not used (low severity) - could create descriptive groups
No feature selection performed (low severity) - let tree-based models handle
Year and Month treated as low-cardinality (low severity) - could encode cyclically if seasonality stronger
